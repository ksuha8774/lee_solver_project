Краткое объяснение для защиты

Я выбрала задачу с волновым алгоритмом Ли, потому что её можно полностью реализовать на Python без внешних сервисов. В программе есть графический интерфейс на Tkinter, загрузка сетки из CSV, выбор начальной и конечной точки, а также сохранение данных в SQL-хранилище SQLite.

Обычный алгоритм Ли распространяет волну по клеткам и находит кратчайший путь, когда все переходы имеют одинаковую стоимость. В моём варианте переходы имеют разные веса: по горизонтали и вертикали шаг равен 1, по диагонали — корень из 2, а нестандартные направления — корень из 5. Кроме того, разные типы поверхности увеличивают стоимость прохода. Поэтому я использовала обобщённый вариант волнового алгоритма — алгоритм Дейкстры с очередью с приоритетом.

В программе также учитываются препятствия. Нельзя переходить в клетку-препятствие, нельзя срезать угол по диагонали через препятствие, а для шага длиной корень из 5 дополнительно проверяется, что на линии движения нет препятствий. Это нужно, чтобы алгоритм не перепрыгивал через непроходимые клетки.

Данные сохраняются в SQLite. В базе есть таблицы для типов поверхности, сеток, клеток, найденных путей и последовательности клеток пути. Поэтому программа не только строит путь визуально, но и сохраняет результат для дальнейшего анализа.
